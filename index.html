A structured full-time curriculum for an STM32 support engineer transitioning into embedded software development, with a specialisation in automotive motor control. Built around your existing debugging depth — adding architecture, RTOS design, control theory, and FOC motor control.

// phases — click to open
PHASE 01
Foundations refresh
6 weeks · 240 hrs · Mon–Sat

Rebuild embedded C from first principles. Layered architecture, HAL design, state machines, memory management, and unit testing habits. Closes the gap between "C that works" and "C you'd show in a code review."

Modern C — Gustedt Embedded SW Design — Beningo C Pointers — finish
PHASE 02
RTOS and real-time design
8 weeks · 320 hrs · Mon–Sat

FreeRTOS internals, queues, semaphores, mutexes, priority inversion, deadlock, DMA integration, LwIP Ethernet, SystemView profiling, and fault handlers. Builds on your existing STM32 + Ethernet support experience.

Hands-On RTOS — Amos Embedded RT OS — K C Wang Making Embedded Systems — White
PHASE 03
Control theory and motor control
10 weeks · 400 hrs · Mon–Sat

Control systems theory (Laplace, PID, Bode), DC motor modelling, BLDC trapezoidal commutation, and full Field-Oriented Control (FOC) implementation on STM32. CAN telemetry and multi-layer fault protection.

Control Systems Eng. — Nise Embedded Control in C — Mann Applied Embedded Elec. — Twomey
PHASE 04
Professional readiness
5 weeks · 200 hrs · Mon–Sat

MISRA-C 2012 compliance, Ceedling+Unity test suites, GitHub Actions CI, portfolio documentation, and deep interview preparation covering embedded C, RTOS, motor control, and system design questions.

Art of Designing ES — Ganssle MISRA-C 2012 Ceedling + Unity + CMock
